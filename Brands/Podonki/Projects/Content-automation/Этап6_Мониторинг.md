# Этап 6 — Мониторинг и аналитика

**Цель**: отслеживать метрики постов (views, forwards, replies), определять что работает, и улучшать систему на основе данных.

## Архитектура

```
Telegram Bot API (getUpdates, getChatHistory)
    ↓
Сбор метрик (views, forwards, replies, reactions)
    ↓
База данных (SQLite + JSON)
    ↓
Анализ (ER, trend detection, A/B results)
    ↓
Отчёт и рекомендации (еженедельно)
    ↓
Feedback в систему (weights для Stage 1)
```

## Сбор метрик

### Методы получения данных

#### 1. Telegram Bot API (getUpdates)
Получить обновления реакций и комментариев:
```
GET https://api.telegram.org/bot{TOKEN}/getUpdates
```

#### 2. TDLib (Desktop API)
Более полные метрики (views, forwards) — требует аккаунта юзера, не бота.

#### 3. Экспорт канала
Периодический экспорт канала через Official Telegram Desktop — парсинг JSON.

#### 4. Polling (простой способ)
Каждый час проверять сообщение:
```
GET https://api.telegram.org/bot{TOKEN}/getChat
```

## Модуль мониторинга (Python)

### Файл: `scripts/мониторинг.py`

```python
#!/usr/bin/env python3
# -*- coding: utf-8 -*-
"""
Мониторинг метрик постов
Сбор views, forwards, replies из Telegram
Расчёт ER и определение тренда
"""

import json
import sqlite3
import sys
from datetime import datetime, timedelta
from pathlib import Path
from typing import Dict, List, Optional
import requests

if sys.platform == 'win32':
    import io
    sys.stdout = io.TextIOWrapper(sys.stdout.buffer, encoding='utf-8')

class MetricsCollector:
    def __init__(self, token: str, db_path: str = 'data/метрики.db'):
        self.token = token
        self.base_url = f"https://api.telegram.org/bot{token}"
        self.db_path = db_path
        self._init_db()

    def _init_db(self):
        """Инициализировать SQLite БД"""
        conn = sqlite3.connect(self.db_path)
        c = conn.cursor()
        c.execute('''
            CREATE TABLE IF NOT EXISTS metrics (
                id INTEGER PRIMARY KEY,
                post_id TEXT UNIQUE,
                channel TEXT,
                message_id INTEGER,
                sent_at TIMESTAMP,
                views INTEGER,
                forwards INTEGER,
                replies INTEGER,
                er REAL,
                collected_at TIMESTAMP
            )
        ''')
        conn.commit()
        conn.close()

    def get_chat_message(self, chat_id: int, message_id: int) -> Dict:
        """
        Получить информацию о сообщении (требует TDLib, но базовая попытка)
        Примечание: Telegram Bot API не возвращает views/forwards напрямую
        """
        # Это ограничение Telegram — можно использовать getUpdates для реакций
        try:
            response = requests.get(f"{self.base_url}/getChat", json={"chat_id": chat_id})
            return response.json()
        except Exception as e:
            return {"error": str(e)}

    def parse_export(self, export_json_path: str) -> List[Dict]:
        """Парсить экспорт канала из Telegram Desktop"""
        with open(export_json_path, 'r', encoding='utf-8') as f:
            data = json.load(f)

        metrics = []
        for msg in data.get('messages', []):
            metric = {
                'message_id': msg.get('id'),
                'text': self._extract_text(msg.get('text', '')),
                'views': msg.get('views', 0),
                'forwards': msg.get('forwards', 0),
                'replies': msg.get('replies', 0),
                'reactions': msg.get('reactions', []),
                'date': msg.get('date')
            }
            metrics.append(metric)

        return metrics

    def _extract_text(self, text_data) -> str:
        """Извлечь текст из структуры"""
        if isinstance(text_data, str):
            return text_data
        if isinstance(text_data, list):
            parts = []
            for item in text_data:
                if isinstance(item, str):
                    parts.append(item)
                elif isinstance(item, dict) and 'text' in item:
                    parts.append(item['text'])
            return ''.join(parts)
        return ''

    def calculate_er(self, forwards: int, replies: int, views: int) -> float:
        """ER = (forwards + replies) / views * 100"""
        if views == 0:
            return 0
        return ((forwards + replies) / views) * 100

    def collect_metrics(self, export_path: str, post_mapping: Dict[int, str]) -> List[Dict]:
        """
        Собрать метрики из экспорта и сопоставить с post_id
        post_mapping: {message_id: post_id}
        """
        metrics = self.parse_export(export_path)
        enriched = []

        for metric in metrics:
            message_id = metric['message_id']
            post_id = post_mapping.get(message_id)

            if post_id:
                er = self.calculate_er(metric['forwards'], metric['replies'], metric['views'])
                enriched.append({
                    'post_id': post_id,
                    'message_id': message_id,
                    'views': metric['views'],
                    'forwards': metric['forwards'],
                    'replies': metric['replies'],
                    'er': round(er, 2),
                    'collected_at': datetime.now().isoformat()
                })

        return enriched

    def save_metrics(self, metrics: List[Dict]):
        """Сохранить метрики в БД и JSON"""
        conn = sqlite3.connect(self.db_path)
        c = conn.cursor()

        for metric in metrics:
            try:
                c.execute('''
                    INSERT OR REPLACE INTO metrics
                    (post_id, message_id, views, forwards, replies, er, collected_at)
                    VALUES (?, ?, ?, ?, ?, ?, ?)
                ''', (
                    metric['post_id'],
                    metric['message_id'],
                    metric['views'],
                    metric['forwards'],
                    metric['replies'],
                    metric['er'],
                    metric['collected_at']
                ))
            except sqlite3.IntegrityError:
                pass

        conn.commit()
        conn.close()

        # Также сохранить в JSON для удобства
        with open('data/метрики_постов.json', 'a', encoding='utf-8') as f:
            for m in metrics:
                f.write(json.dumps(m, ensure_ascii=False) + '\n')

def analyze_metrics(db_path: str) -> Dict:
    """Анализировать метрики из БД"""
    conn = sqlite3.connect(db_path)
    c = conn.cursor()

    # Получить все метрики за последние 7 дней
    week_ago = (datetime.now() - timedelta(days=7)).isoformat()
    c.execute('''
        SELECT er, content_type, rubric FROM metrics
        WHERE collected_at > ?
    ''', (week_ago,))

    rows = c.fetchall()
    conn.close()

    if not rows:
        return {"error": "No data"}

    ers = [row[0] for row in rows]
    avg_er = sum(ers) / len(ers)

    # Найти лучшие рубрики (по среднему ER)
    rubric_ers = {}
    for row in rows:
        rubric = row[2]
        if rubric not in rubric_ers:
            rubric_ers[rubric] = []
        rubric_ers[rubric].append(row[0])

    top_rubrics = sorted(
        rubric_ers.items(),
        key=lambda x: sum(x[1]) / len(x[1]),
        reverse=True
    )

    return {
        "period": "last_7_days",
        "avg_er": round(avg_er, 2),
        "total_posts": len(ers),
        "top_rubrics": [(r[0], round(sum(r[1]) / len(r[1]), 2)) for r in top_rubrics[:5]]
    }

def main():
    # Загрузить токен
    with open(Path.home() / '.claude' / 'api-keys' / 'telegram-bot-token', 'r') as f:
        token = f.read().strip()

    collector = MetricsCollector(token)

    # Загрузить mapping message_id → post_id
    with open('data/публикация_логи.json', 'r', encoding='utf-8') as f:
        logs = [json.loads(line) for line in f]
        post_mapping = {log['message_id']: log['id'] for log in logs if log.get('message_id')}

    # Собрать метрики из экспорта
    # (предполагаем, что экспорт уже есть в data/экспорт_канала.json)
    metrics = collector.collect_metrics('data/экспорт_канала.json', post_mapping)

    # Сохранить
    collector.save_metrics(metrics)

    # Анализировать
    analysis = analyze_metrics(collector.db_path)
    print(json.dumps(analysis, ensure_ascii=False, indent=2))

    # Сохранить отчёт
    with open('data/анализ_неделя.json', 'w', encoding='utf-8') as f:
        json.dump(analysis, f, ensure_ascii=False, indent=2)

    print("✅ Метрики собраны и проанализированы")

if __name__ == '__main__':
    main()
```

### Как применить feedback:
1. Загрузить feedback в `config/обратная_связь.json`
2. На Этап 1 используются эти weights при генерации идей
3. Идеи с высокой ER получают бонус к score
4. Минимизируются идеи, которые исторически показали низкую ER

## Инструменты для мониторинга

### 1. TG-Desktop экспорт (вручную)
- Открыть канал → Меню → Экспорт истории
- Сохранить как JSON
- Обработать скриптом

### 2. n8n automation (рекомендуется)
```
Trigger: Weekly (по понедельникам в 09:00)
    ↓
Get latest messages from channel (manual экспорт или polling)
    ↓
Call мониторинг.py
    ↓
Generate report JSON
    ↓
Send summary to Telegram (админу)
```

### 3. Google Sheets (для просмотра)
- Синхронизировать data/метрики_постов.json в Google Sheets
- Построить графики ER по времени
- Визуализация по рубрикам

## Возможные проблемы и решения

| Проблема | Причина | Решение |
|----------|---------|---------|
| Нет views/forwards в Bot API | Ограничение Telegram | Использовать экспорт Desktop или TDLib |
| Данные собираются с задержкой | Кеширование Telegram | Собирать метрики через 6+ часов после публикации |
| Несоответствие метрик | Разные источники данных | Использовать один источник (Desktop export) |
| Старые посты теряют views | История удаляется | Архивировать метрики каждый день |

## Следующие шаги

1. **Настроить Telegram Desktop экспорт** — еженедельно вручную
2. **Запустить мониторинг.py** — обрабатывать экспорт
3. **Собрать feedback** — за неделю-две данных
4. **Обновить weight в Этап 1** — на основе feedback
5. **Повторить цикл** — улучшение через данные
